type definition 2

타입 정의를 조금 더 고급스럽게 사용하는 법을 알아보자.

1. type 과 interface 같이 자식을 포함할 수 있는 타입들의 병합

- & 연산자

두 타입을 한 타입으로 합치는 연산자다. 중복되는 프로퍼티는 &(교집합) 처리가 되는데 프로퍼티가 서로 다른 타입일 경우 string & number 같이 정의되어 값을 할당할 수 없게 되니 조심해야 한다. 합쳐진 타입 안에는 이전의 모든 프로퍼티가 존재하게 되므로 모두 assist 가 가능하다. 따라서 두 타입의 프로퍼티가 서로 완전 다를 때 사용하는 것이 바람직하다.

- | 연산자

둘 중 한 타입에만 속하면 되는 연산자다. 범용적으로 두 타입을 합치는 효과가 있다. & 연산자와는 다르게 중복되는 프로퍼티는 타입끼리 |(합집합) 처리가 되어 안정적으로 값을 할당할 수 있다. 하지만 어느 한 타입에만 존재하는 프로퍼티가 있을 수 있어 assist 는 중복되는 프로퍼티만 가능하다. 따라서 적절한 type assertion 을 통해 효율적으로 사용하는 것이 좋겠다.

2. extends

extends 는 제네릭 안에서 어떤 타입(들)의 선택적 포함을 나타낸다.

// T 는 "address1" | "address2"
declare class IPerson<T extends "address1" | "address2"> {
  name: string;
  age: number;
  address: T
}

3. keyof

keyof 는 어떤 타입의 프로퍼티를 Union 타입으로 나열하는 키워드로써 제네릭이나 타입을 정의하는데 효율적으로 사용된다.

declare namespace Info {
  class Address {
    address1: string;
    address2: string;
  }

  // T 는 "address1" | "address2"
  interface IPerson<T extends keyof Address> {
    name: string;
    age: number;
    address: T;
  }

  type TPerson = {
    name: string,
    age: number,
    address: string
  }

  // unionTPerson 은 "name" | "age" | "address"
  export const unionTPerson: keyof TPerson;
}

4. in

in 은 어떤 타입(들)의 포함을 나타낸다.

declare namespace Info {
  type Partial<T> = {
    [K in keyof T]?: T[K]
  }
  
  interface IRequiredProperty {
    name: string;
    age: number;
    address: string;
  }
  
  // 모든 프로퍼티를 선택적인 프로퍼티로 변경
  type ISelectedProperty = Partial<IRequiredProperty>;

  // 필수적인 "name", "age", "address" 프로퍼티
  type Person = {
    [T in "name" | "age" | "address"]: string
  }
}

5. 대표적인 객체 타입 라이브러리

// 모든 프로퍼티를 readonly 프로퍼티로 변경
type Readonly<T> = {
    readonly [P in keyof T]: T[P];
}

// 모든 프로퍼티를 선택적인 프로퍼티로 변경
type Partial<T> = {
    [P in keyof T]?: T[P];
}

// 모든 프로퍼티를 null 허용 프로퍼티로 변경
type Nullable<T> = {
    [P in keyof T]: T[P] | null;
}

// 모든 프로퍼티 중 선택한 일부 프로퍼티로 변경
type Pick<T, K extends keyof T> = {
    [P in K]: T[P];
}

// Record
type Record<K extends string | number, T> = {
    [P in K]: T;
}
