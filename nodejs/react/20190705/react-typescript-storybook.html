React + TypeScript + StoryBook

1. 프로젝트 만들기

create-react-app 을 통해 TypeScript 기반 프로젝트 만들기

npx create-react-app my-app --typescript


2. StoryBook 추가하기

1. 프로젝트 폴더

cd my-app

2. 설치하기

npx -p @storybook/cli sb init

3. typings

npm install --save-dev @types/storybook__react

4. Webpack Config 추가

.storybook 폴더 안에 webpack.config.js 를 추가한다.

// webpack.config.js
module.exports = ({ config, mode }) => {
  config.module.rules.push({
    test: /\.(ts|tsx)$/,
    loader: require.resolve('babel-loader'),
    options: {
      presets: [['react-app', { flow: false, typescript: true }]],
    },
  });
  config.resolve.extensions.push('.ts', '.tsx');
  return config;
};

5. 확장자 변경

StoryBook 을 설치하면서 생성된 파일들을 TypeScript 프로젝트에 맞게 확장자를 변경해 보자. *.stories.tsx 확장자는 다음에 config.ts 를 수정하면서 StoryBook 에 불러오는 조건으로 매칭시킬 것이다.

// .storybook 폴더
addons.js > addons.ts
config.js > config.ts

// src/stories 폴더
index.js > index.stories.tsx

6. config.ts 수정

  • .storybook 폴더에 위치
  • src 폴더를 전체 범위로 할 것
  • *.stories.tsx 확장자를 가진 파일들만 StoryBook 에 불러올 것

StoryBook 에 불러오는 조건을 TypeScript 프로젝트에 맞게 더 advanced 하게 변경한 것이다.

import { configure } from '@storybook/react';

// *.stories.tsx 확장자를 가진 파일들만 자동으로 import
const req = require.context('../src', true, /\.stories\.tsx$/);

function loadStories() {
  req.keys().forEach(req);
}

configure(loadStories, module);

7. StoryBook 독립적으로 사용하기 (기본설정)

기본적으로 stories 폴더는 src 안에 있고 독립적으로 사용하게 config 되어 있다. 혹시나 stories 폴더를 src 외부에 놓고 싶다면 아래와 같이 tsconfig.json 과 config.ts 를 수정해 준다.

- tsconfig.json

{
  "compilerOptions": {
    "rootDirs": ["src", "stories"]
  }
}

- config.ts

*.stories.tsx 확장자를 가진 파일들만 StoryBook 에 불러올 것

import { configure } from '@storybook/react';

// *.stories.tsx 명을 가진 파일들을 자동으로 import
const req = require.context('../stories', true, /\.stories\.tsx$/);

function loadStories() {
  req.keys().forEach(req);
}

configure(loadStories, module);
