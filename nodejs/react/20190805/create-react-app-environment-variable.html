create-react-app 에서 환경변수 관리

1. ‘NODE_ENV’ 환경변수

NODE_ENV 환경변수는 특수한 환경변수로 개발자가 설정할 수 없으며 설정해도 무시된다. package.json 의 script 에 따라 자동으로 값이 대입되는데, npm start 는 development 를, npm run build 는 production 을, npm test 는 test 를 대입한다.

2. 변수명은 반드시 ‘REACT_APP_’ 으로 시작되어야 한다.

create-react-app 에서는 보안이 필요한 환경변수의 유출을 미연에 방지하기 위해 REACT_APP_ 으로 시작되지 않는 환경변수는 무시한다. 예를 들면 한 파일에 백엔드와 함께 환경변수를 설정했을 경우, API_KEY 나 SECRET_KEY 가 유출될 가능성을 배제할 수 없다. 더 자세한 이유는 3.에서 설명한다.

3. 보안이 필요한 값은 환경변수로 사용하지 않는다.

사용된 환경변수는 번들링 중에 해당 값으로 교체된다. 번들 된 파일들이 static 적으로 참조되기 때문이다. 그래서 보안이 필요한 API_KEY 나 SECRET_KEY 를 환경변수로 사용할 경우, 번들 된 파일 안에 값들이 그대로 노출되는 초유의 사태가 벌어질 수 있다. 따라서 이 점을 유의하며 환경변수를 설정하도록 하자.

4. html 에서 환경변수 사용

html 에서도 번들링 중에 환경변수를 해당 값으로 교체할 수 있다.

  <title>%REACT_APP_TITLE%</title>

5. .env 파일 결정 및 우선 순위

pacakge.json 의 script 에 따라 불러오는 .env 파일이 결정된다. 왼쪽으로 갈수록 우선 순위가 높다.

  • npm start : .env.development.local, .env.development, .env.local, .env
  • npm run build : .env.production.local, .env.production, .env.local, .env
  • npm test : .env.test.local, .env.test, .env
