create-react-app 에서 absolute path 와 alias path 사용하기

Project Environment

  • create-react-app –typescript
  • react-app-rewired (override webpack config without ejecting)
  • storybook
  • tslint


1. create-react-app

create-react-app 으로 생성한 프로젝트는 NODE_PATH 환경변수만 설정해도 absolute path 를 사용할 수 있다. 이 환경변수는 webpack config 의 resolve.modules 에 추가되는 값으로써 webpack.config.js 를 직접 수정해도 무방하다. 하지만 alias path 를 사용하려면 webpack.config.js 를 직접 수정하는 방법밖에 없다.

1. build process

create-react-app –typesciprt 로 생성한 프로젝트는 *.ts, *.tsx 파일들로 구성되있다. 먼저, TypeScript Compiler 가 이 파일들을 *.js, *.jsx 로 컴파일 하면, Webpack 이 다시 이 파일들을 concat, minify 처리하여 bundle.js 로 만든다. 따라서 absolute path 와 alias path 를 사용하기 위해서는 tsconfig.json 과 webpack.config.js 를 둘 다 수정해야 한다.

*.ts, *.tsx → *.js, *.jsx → bundle.js

tsconfig.json 에 설정하는 값은 단순히 TypeScript Compiler 에게 알려주기 위함이고 실질적으로 Webpack 이 webpack.config.js 를 통해 absolute path 와 alias path 를 처리(transpile)한다.

2. absolute path

create-react-app 에서는 기본적으로 ‘ . ‘ 값을 설정하지 못하도록 되있다.

- tsconfig.json

compilerOptions 에 baseUrl 값을 설정한다.

{
  "compilerOptions": {
    "baseUrl": "src"
  }
}

- webpack.config.js

config 의 resolve.modules 안에 설정값을 추가한다.
webpack.config.js 를 수정하는 대신에 NODE_PATH 환경변수를 설정해도 역시나 동일한 설정이다.

// config-overrides.js
const path = require('path');

module.exports = (config, env) {
  ...
  config.resolve.modules = [
    ...(config.resolve.modules || []),
    path.resolve(__dirname, 'src')  // 추가
  ];
  ...
};

위에 코드는 react-app-rewired 모듈을 통해 eject 없이 webpack config 를 overriding 하는 것으로, 기존 object 형태의 설정과 다를 수 있다. 하지만 config structure 는 동일하므로 설정값을 쉽게 추가할 수 있을 것이다.

3. alias path

absolute path 를 alias 와 함께 사용한다면 코드의 가독성을 높일 수 있다. 예를 들면 ‘components/Button’ 을 ‘@components/Button’ 으로 작성이 가능하다. 위에서 설정한 baseUrl 기준으로 경로를 매핑시켜 보겠다.

- tsconfig.json

compilerOptions 에 paths 값을 설정한다.

{
  "compilerOptions": {
    "baseUrl": "src",
    "paths": {
      "@components*": ["components*"]
    }
  }
}

- webpack.config.js

config 의 resolve.alias 안에 paths 와 싱크된 값을 추가한다. 두 값에 살짝 차이가 있다는 점을 유의하자.

// config-overrides.js
const path = require('path');

module.exports = (config, env) {
  ...
  config.resolve.modules = [
    ...(config.resolve.modules || []),
    path.resolve(__dirname, 'src')
  ];

  config.resolve.alias = {
    ...(config.resolve.alias || {}),
    '@components': 'components' // 추가
  }
  ...
};

2. storybook

storybook 도 Webpack 을 사용하기 때문에 위와 완전히 동일하다. 이미 tsconfig.json 을 설정했으므로 .storybook 폴더 안에 webpack.config.js 만 수정하면 된다. 다른게 하나 있다면 파일 위치가 루트 폴더가 아니라서 ../src 값을 가지고 있다.

const path = require('path');

module.exports = ({ config, mode }) => {
  ...
  config.resolve.modules = [
    ...(config.resolve.modules || []),
    path.resolve(__dirname, '../src')
  ];

  config.resolve.alias = {
    ...(config.resolve.alias || {}),
    '@components': 'components'
  };
  ...
};

3. tslint

linting 에 alias path 가 걸리지 않도록 no-implicit-dependencies rule 을 추가한다.

{
  "rules": {
    "no-implicit-dependencies": [false, ["@components"]]
  }
}
